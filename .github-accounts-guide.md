# Managing Multiple GitHub Accounts in Cursor

This guide helps you use 2 different GitHub accounts for different projects/workspaces in Cursor.

## Method 1: Per-Repository Git Config (Simplest)

Each repository can have its own Git user configuration, independent of your global settings.

### Setup for Current Repository (LuxAI)

```bash
# For Account 1 (SuperThinks)
git config --local user.name "SuperThinks"
git config --local user.email "superthinksai@gmail.com"

# OR for Account 2 (if different)
git config --local user.name "YourOtherAccountName"
git config --local user.email "your-other-email@example.com"
```

### For Other Repositories

When you open a different repository in Cursor:
1. Open terminal in that repository
2. Run the same commands with the appropriate account details

### Verify Configuration

```bash
# Check global config
git config --global --list | grep user

# Check local config (for current repo)
git config --local --list | grep user
```

## Method 2: SSH Keys (Best for Multiple Accounts)

This method uses different SSH keys for different GitHub accounts.

### Step 1: Generate SSH Keys for Each Account

```bash
# For Account 1 (SuperThinks)
ssh-keygen -t ed25519 -C "superthinksai@gmail.com" -f ~/.ssh/id_ed25519_account1

# For Account 2
ssh-keygen -t ed25519 -C "your-other-email@example.com" -f ~/.ssh/id_ed25519_account2
```

### Step 2: Add SSH Keys to GitHub

1. Copy the public key:
   ```bash
   cat ~/.ssh/id_ed25519_account1.pub | pbcopy
   ```

2. Go to GitHub → Settings → SSH and GPG keys → New SSH key
3. Paste and save
4. Repeat for account 2

### Step 3: Configure SSH Config

Create/edit `~/.ssh/config`:

```ssh-config
# Account 1 (SuperThinks) - for AaraikAI organization
Host github.com-account1
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_account1
    IdentitiesOnly yes

# Account 2 - for personal projects
Host github.com-account2
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_account2
    IdentitiesOnly yes
```

### Step 4: Update Remote URLs

For repositories using Account 1:
```bash
git remote set-url origin git@github.com-account1:AaraikAI/LuxAI.git
```

For repositories using Account 2:
```bash
git remote set-url origin git@github.com-account2:username/repo.git
```

## Method 3: Git Credential Helper with Different Stores

Use different credential stores for different accounts.

### Setup

```bash
# For Account 1 repositories
git config --local credential.helper 'store --file ~/.git-credentials-account1'

# For Account 2 repositories  
git config --local credential.helper 'store --file ~/.git-credentials-account2'
```

Then when you push, enter the appropriate Personal Access Token for each account.

## Method 4: Conditional Git Config Includes (Advanced)

Use Git's conditional includes based on directory.

### Setup

1. Create config files:
   ```bash
   # ~/.gitconfig-account1
   [user]
       name = SuperThinks
       email = superthinksai@gmail.com
   
   # ~/.gitconfig-account2
   [user]
       name = YourOtherAccountName
       email = your-other-email@example.com
   ```

2. Edit `~/.gitconfig`:
   ```ini
   [includeIf "gitdir:~/Desktop/AARAIK LLC/"]
       path = ~/.gitconfig-account1
   
   [includeIf "gitdir:~/PersonalProjects/"]
       path = ~/.gitconfig-account2
   ```

## Recommended Approach for Cursor Profiles

Since you mentioned "2 profiles in Cursor", I recommend:

1. **Use Method 1 (Per-Repository Config)** for simplicity
2. **Use Method 2 (SSH Keys)** if you want better security and automatic authentication

## Quick Setup Script

Run this in each repository to set it up:

```bash
# For Account 1
./setup-git-account.sh account1 "SuperThinks" "superthinksai@gmail.com"

# For Account 2
./setup-git-account.sh account2 "YourOtherAccountName" "your-other-email@example.com"
```

## Troubleshooting

### Check which account is being used:
```bash
git config user.name
git config user.email
```

### Check remote URL:
```bash
git remote -v
```

### Test SSH connection:
```bash
ssh -T git@github.com-account1
ssh -T git@github.com-account2
```
